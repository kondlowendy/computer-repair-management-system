# Kanban Board Explanation

## What is a Kanban Board?

A Kanban board is a visual project management tool used to track tasks as they move through different stages of a workflow. It uses columns such as "To Do", "In Progress", and "Done" to represent the progress of tasks.

---

## How the Board Visualizes Workflow

The Kanban board created for the Computer Repair Management System includes the following columns:

- Backlog: Tasks that are planned but not yet started
- To Do: Tasks ready to be worked on
- In Progress: Tasks currently being developed
- Testing: Tasks being tested for quality assurance
- Blocked: Tasks that cannot proceed due to dependencies
- Done: Completed tasks

Each task (issue) moves from left to right, showing its progress clearly.

---

## Work-in-Progress (WIP) Limits

WIP limits were applied to control the number of tasks in certain columns. For example:

- In Progress (Max: 3)
- Testing (Max: 2)

This prevents overload and ensures that tasks are completed before new ones are started, improving efficiency and focus.

---

## Support for Agile Principles

The Kanban board supports Agile principles in several ways:

- Continuous delivery: Tasks are completed and moved to "Done" regularly
- Flexibility: Tasks can be reprioritized as needed
- Transparency: Team members can see task status at any time
- Collaboration: Tasks are assigned and tracked clearly

Additionally, the use of labels such as "feature", "bug", and priority levels helps organize work and improve decision-making.

---

## Conclusion

The Kanban board effectively manages the workflow of the system by providing visibility, limiting work overload, and supporting Agile development practices.
