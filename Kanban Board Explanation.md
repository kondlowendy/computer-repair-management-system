# Kanban Board Explanation

A Kanban board is a visual project management tool used to track tasks as they move through different stages of a workflow. It helps teams understand the current status of work, identify bottlenecks, and manage progress efficiently.

## Workflow Visualization

The Kanban board created for the Computer Repair Management System uses the following columns:

Backlog, To Do, In Progress, Testing, Blocked, and Done.

Each column represents a stage in the system development process.

- Backlog contains all planned development tasks that have not yet been started.
- To Do includes tasks selected for the current sprint or development cycle.
- In Progress shows tasks that are actively being worked on.
- Testing represents tasks that are completed but undergoing verification or quality assurance.
- Blocked includes tasks that cannot proceed due to dependencies or issues.
- Done contains tasks that have been successfully completed.

This structure allows clear visualization of how tasks move from initial planning to completion.

## Work-In-Progress (WIP)

The Kanban board helps limit work-in-progress by controlling how many tasks are placed in the “In Progress” column at a time. This ensures that developers focus on completing tasks before starting new ones, reducing overload and improving efficiency.

## Support for Agile Principles

The Kanban board supports Agile methodology in several ways. It enables continuous delivery by allowing tasks to move through the workflow as they are completed. It also supports adaptability, as tasks can be reprioritized or moved between columns when necessary. In addition, it improves collaboration by clearly showing task ownership and progress.

## Conclusion

Overall, the Kanban board provides a structured and visual way to manage system development tasks. It improves visibility, supports efficient workflow management, and aligns with Agile practices, making it suitable for developing the Computer Repair Management System.
