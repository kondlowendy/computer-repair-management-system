# Computer Repair Management System - User Stories

| ID | User Story | Priority |
|----|------------|----------|
| US-01 | As a customer, I want to register my device so that it can be booked for repair. | High |
| US-02 | As a technician, I want to record device intake so that I can track received devices and accessories. | High |
| US-03 | As a technician, I want to add accessories to a device so that all items received are properly documented. | Medium |
| US-04 | As a technician, I want to update repair status so that customers can track progress. | High |
| US-05 | As a customer, I want to track my repair status so that I know when my device will be ready. | High |
| US-06 | As a customer, I want to choose a service option (pickup/delivery/drop-off) so that I can receive convenient service. | Medium |
| US-07 | As an admin, I want to calculate profit so that I can monitor business performance. | Medium |
| US-08 | As an admin, I want to view reports so that I can analyze repair activities and revenue. | Medium |
| US-09 | As a delivery staff member, I want to track device status so that I can manage deliveries efficiently. | Low |
| US-10 | As a system, I want to send notifications so that customers are informed about repair updates. | Medium |
