# Reflection

## Challenges

One challenge was deciding the level of detail for states and actions. Too many states made diagrams complex and hard to read, while too few reduced clarity.

## Alignment with Agile

Aligning diagrams with user stories required reviewing Assignment 6 to ensure consistency between workflows and system functionality.

## State vs Activity Diagrams

State Diagrams:

* Show object lifecycle and state changes over time

Activity Diagrams:

* Show workflows and processes

State diagrams helped model system behavior, while activity diagrams helped visualize operations and interactions.

## Lessons Learned

This assignment improved understanding of system dynamics and how different components interact within a real-world application.
