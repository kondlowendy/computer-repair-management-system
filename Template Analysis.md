# Template Analysis – GitHub Project Templates

GitHub provides several project templates that can be used to manage software development workflows. These include Basic Kanban, Bug Triage, Team Planning, and the Board template available in GitHub Projects. Each template is designed for a different purpose and supports different types of workflows.

## Basic Kanban

The Basic Kanban template is the simplest option. It typically includes columns such as “To Do,” “In Progress,” and “Done.” While it is easy to use and suitable for small projects, it lacks the flexibility required for more complex system development. It does not support additional stages such as testing or handling blocked tasks, which are important in real-world development environments.

## Bug Triage

The Bug Triage template is designed specifically for managing and prioritizing software bugs. It includes stages such as “New,” “Needs Review,” and “In Progress.” Although it is useful for identifying and resolving issues, it is not suitable for managing full system development because it focuses only on problem resolution rather than feature development.

## Team Planning

The Team Planning template is more advanced and supports backlog management and task prioritization. It is useful for organizing work before development begins and aligns well with sprint planning. However, it still does not fully represent all stages of system development, particularly testing and handling blocked tasks.

## Selected Template: Board (GitHub Projects)

The Board template in GitHub Projects was selected for this assignment. This template allows full customization of columns and integrates directly with GitHub Issues. It supports a complete development workflow by allowing tasks to move through stages such as Backlog, To Do, In Progress, Testing, Blocked, and Done.

## Justification

The ability to add custom columns such as “Testing” and “Blocked” was particularly important. The Testing column represents the quality assurance phase, while the Blocked column helps track tasks that cannot progress due to dependencies or issues. These stages are essential in real-world software development and are not supported in simpler templates.

In addition, the Board template supports Agile principles by enabling continuous task movement, clear visualization of progress, and better collaboration through task assignment and issue tracking. It also provides traceability, as each task can be linked to a GitHub Issue and assigned to a developer.

## Conclusion

The Board template is the most suitable option for this project because it supports a full development lifecycle, allows customization of workflow stages, and integrates well with Agile practices. It provides better control and visibility compared to other templates, making it ideal for managing the development of the Computer Repair Management System.
