# Template Analysis and Selection

## Comparison of GitHub Project Templates

| Template | Columns/Workflow | Automation Features | Agile Suitability |
|---------|----------------|---------------------|------------------|
| Basic Kanban | To Do, In Progress, Done | None | Suitable for simple workflows but lacks automation |
| Automated Kanban | To Do, In Progress, Done | Auto-moves issues when status changes | Highly suitable for Agile as it supports continuous workflow |
| Bug Triage | New, Needs Review, Closed | Helps manage incoming bugs | Useful for maintenance but not full project tracking |
| Team Planning | Backlog, Ready, In Progress, Done | Supports planning and prioritization | Suitable for sprint planning but less flexible |

---

## Selected Template: Automated Kanban

The **Automated Kanban** template was selected for this project because it aligns well with Agile development principles. It allows tasks (issues) to automatically move between columns based on their status, reducing manual work and improving workflow efficiency.

This template supports:
- Continuous task tracking
- Real-time updates
- Clear visualization of progress

---

## Justification

The Computer Repair Management System requires tracking multiple tasks such as device intake, repair processing, and reporting. The Automated Kanban template supports this by:

- Allowing tasks to flow smoothly between stages
- Supporting iterative development
- Improving team productivity through automation

Additionally, it was flexible enough to allow customization, such as adding new columns like **Testing** and **Blocked**, which are important for quality assurance and dependency management.

Therefore, the Automated Kanban template was the most suitable choice for this project.
