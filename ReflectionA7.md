# Reflection

One of the main challenges in this assignment was selecting the most appropriate GitHub project template. Initially, it was difficult to decide between templates such as Basic Kanban and Automated Kanban, as both seemed suitable. However, after comparing their features, it became clear that Automated Kanban provided better support for Agile workflows due to its automation capabilities.

Another challenge was customizing the Kanban board. Adding new columns such as "Testing" and "Blocked" required careful thought to ensure they aligned with the development process. It was also challenging to correctly classify tasks as features or bugs, as this required understanding the difference between new functionality and system errors.

Managing task prioritization was also a learning experience. Assigning priority levels helped in understanding which parts of the system were most critical, such as device intake and pricing logic.

When comparing GitHub Projects to tools like Jira and Trello, GitHub was easier to integrate with code and issues, making it very convenient for developers. However, Jira offers more advanced features for large teams, and Trello provides a more user-friendly interface.

Overall, this assignment helped me understand how Kanban boards support Agile development, improve workflow visibility, and enhance project organization. It also gave me practical experience using GitHub Projects, which is valuable for real-world software development.
