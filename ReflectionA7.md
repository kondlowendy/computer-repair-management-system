# Reflection

This assignment provided practical insight into how Agile project management tools, such as GitHub Projects, can be used to organize and manage system development tasks. One of the main challenges encountered was selecting the most appropriate project template. GitHub offers multiple templates, including Basic Kanban, Bug Triage, and Team Planning, each designed for different purposes. Initially, it was not clear which template would best suit the development of the Computer Repair Management System. After analysis, it became evident that some templates, such as Bug Triage, focus mainly on issue resolution rather than full system development, making them unsuitable for this project.

Another challenge was customizing the Kanban board to reflect a realistic development workflow. The default templates did not fully represent all stages required for building the system. For example, software development involves not only coding but also testing and handling blocked tasks. To address this, additional columns such as "Testing" and "Blocked" were introduced. This required a deeper understanding of how real-world development teams structure their workflows and manage dependencies between tasks.

A further difficulty was ensuring that the tasks placed on the board were development-focused rather than user-focused. Earlier assignments emphasized user requirements and use cases, but this assignment required translating those into actual development tasks, such as implementing backend modules, designing database schemas, and building reporting functionality. This shift in perspective was initially challenging but ultimately helped in understanding how requirements evolve into implementation.

Trello is user-friendly and ideal for simple task tracking, while Jira offers advanced features for large-scale enterprise projects. However, GitHub Projects stands out because of its direct integration with repositories, issues, and development workflows. This makes it particularly useful for software development projects, as it connects planning and implementation in one platform.

Overall, this assignment improved my understanding of Agile methodologies, particularly the use of Kanban boards to visualize workflow and manage tasks. It also highlighted the importance of selecting appropriate tools and customizing them to fit the needs of a specific project. The experience gained will be valuable in future software development projects, especially in collaborative and iterative environments.
