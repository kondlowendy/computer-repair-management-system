## Kanban Board Customization

The Kanban board was customized to support system development for the Computer Repair Management System.

Two columns were added:

- Testing: used to verify completed development tasks before marking them as done.
- Blocked: used to track tasks that cannot continue due to issues or dependencies.

The final workflow is:
Backlog → To Do → In Progress → Testing → Blocked → Done

GitHub Issues were used to represent development tasks and were linked to the board. Labels such as "feature" were applied, and tasks were assigned using @mentions.

These customizations improve task tracking, visibility, and support Agile development practices.
