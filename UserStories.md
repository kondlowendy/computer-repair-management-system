# Computer Repair Management System - User Stories

| ID | User Story | Priority (MoSCoW) |
|----|------------|------------------|
| US-01 | As a customer, I want to register my device so that it can be booked for repair. | Must-have |
| US-02 | As a technician, I want to record device intake so that I can track received devices and accessories. | Must-have |
| US-03 | As a technician, I want to add accessories to a device so that all items received are properly documented. | Should-have |
| US-04 | As a technician, I want to update repair status so that customers can track progress. | Must-have |
| US-05 | As a customer, I want to track my repair status so that I know when my device will be ready. | Must-have |
| US-06 | As a customer, I want to choose a service option (pickup/delivery/drop-off) so that I can receive convenient service. | Should-have |
| US-07 | As an admin, I want to calculate profit so that I can monitor business performance. | Should-have |
| US-08 | As an admin, I want to view reports so that I can analyze repair activities and revenue. | Could-have |
| US-09 | As a delivery staff member, I want to track device status so that I can manage deliveries efficiently. | Won’t-have |
| US-10 | As a customer, I want to receive notifications so that I stay updated on my repair status. | Could-have |
